Mind Map: Unified ID Logic for Training Data

Training Session ID
(THE CENTRAL HUB)

1. Training Session Reporting Form (Trainer)

  • Purpose: Generate unique Training Session ID
  • Key Data: Session Details, Beneficiary Counts, Topics Covered
  • Automatic Captures: GPS, Audio Recording

2. Beneficiary Details Form (Data Recorder)

  • Purpose: Individual beneficiary data
  • Linkage: Requires **Training Session ID**
  • Key Data: CNIC, Pictures, Contact Info, Financial Data

3. IC Monitoring Form (IC)

  • Purpose: On-the-ground, third-party verification
  • Linkage: Requires **Training Session ID**
  • Key Data: Venue Assessment, Beneficiary Spot-Check, Trainer Rating
  • Automatic Captures: GPS

4. Quality Assurance (QA) Form (QA Team)

  • Purpose: Remote audit of content and delivery
  • Linkage: Observes a training linked by **Training Session ID**
  • Key Data: Concepts Covered Checklist, Observations/Notes

5. Recording Review Team Form (Audio Auditor)

  • Purpose: Audit session via audio recording
  • Linkage: Audits the audio file linked by **Training Session ID**
  • Key Data: Audio Quality, Content Review, Beneficiary Engagement
